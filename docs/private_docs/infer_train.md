# 🚩 推理训练设计文档

---

首先明确当前智能体的相关输入输出。
实际上是一个路径规划任务。

智能体基本输出应该是每一循环上的决策，实际上是包含时序在内的。
因此只需要实现智能体每一循环给出输出即可完成对应的推理任务。

当前需要了解如何在系统内执行定时任务，即无关循环步，仅关注env_time等指标。

智能体的训练，可以参考当前路径规划的进展。

近几年工厂/仓储级别的 AGV 路径规划（特别是多车协同、多任务调度场景）既受到经典算法优化的推动，
也被机器学习、图神经网路、以及“数字孪生 + 边缘计算/5G”等工程实践推动，出现了“理论→可扩展实现→工业落地”三条并行推进的趋势。

1. 多智能体路径规划（MAPF）与 Conflict-Based Search（CBS）家族

> CBS 及其变体仍然是工业多车冲突消解的主流框架（优点：清晰分层（高层冲突分裂 / 低层单车搜索），容易嵌入约束和代价函数）。
> 研究在于提升可扩展性与引入应用层约束（如优先级、交通规则、充电点等）。
> (a) 在保证/接近最优的前提下做剪枝与启发式改进（如 bounded/ improved CBS）；
> (b) 并行化与硬件加速（例如 GPU 加速的 CBS），以应对上百/上千车的在线规划需求。
> 工程上这些改进直接决定能否在工厂级 fleet 中实时运行。

2. 图神经网络（GNN）+ 强化学习（RL）／多智能体深度学习（MADRL）
   学界近两年大量工作把 MAPF 看成图上决策问题，使用 GNN 提取局域/全局交互表示，
   再结合集中式或分布式 RL 做决策/策略学习（优势：能隐式学习协同规则，鲁棒于动态障碍与不确定性；
   缺点：训练/泛化与可解释性仍是瓶颈）。

3. 端到端 RL 与混合方法（经典规划 + 学习）

## 评估指标（工程/研究都在看）

经典：总路径长度 / 总时间 / makespan / 能耗 / 任务完成率。
实时场景还看：在线响应时间、冲突恢复时间、充电调度成功率、以及系统级吞吐（jobs/hour）。
（这些指标直接决定一个算法能否在工厂里被接受；论文通常在这些指标上做对比。）

## 给工程团队/研究者的建议（如何落地或继续研究）

若你目标是落地部署：优先选择“经典 MAPF/CBS（或其工程化变体）+ 局部避障（基于模型或RL）+ 数字孪生仿真闭环”
的路线；同时准备把求解器并行化或分区（区域化调度）以保证实时性。

若你是研究/论文方向：可以聚焦（a）GNN+MADRL 在信息受限的分布式协同（如何保证收敛与稳定）；（b）可解释/安全的学习-规划混合框架；或（c）大规模
GPU/CUDA 加速的优化器（例如 GPU-CBS）。

工具与数据：搭建高保真数字孪生环境做大量仿真（含感知误差与能耗模型），再用仿真数据做 RL 预训练，最后做小规模实车验证（sim-to-real）。
