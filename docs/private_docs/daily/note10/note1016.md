

实现一个并行机+LMAPF的模型，模型描述如下：

网格化环境（离散时间）用于 AGV（小车）移动与避让（基于优先级规划与时隙保留表的简单实现）。
多台机器（Machine），每台机器有位置与可用时间。
若干 jobs：包含 pickup（取料点）和 processing time（处理时间）。
一个贪心分配器：将每个 job 分配到使预计完成时间最小的机器（同时考虑运输距离的启发式估算）。
AGV 路径使用 A* 做空间路径，并用简单的等待插入与保留表避免冲突（时间扩展的优先级规划风格）。
离散步进仿真，记录时间线与最终作业状态（可视化为表格）。

实现一个并行机处理的任务，和当前的machine结合一下。当前工厂的实现实际上没有machine。


在普通的并行机调度中，我们只考虑：
Cj=Sj+pj
即完成时间=开始时间+处理时间

加入运输环节后，作业的实际开始时间取决于物料到达时间。
Sj=max(Aj,Tarrive)

因此调度层决定job的分配，MAPF曾觉得AGV如何在工厂网格上移动运输物料。
故形成强耦合。

核心难点：
    任务分配依赖路径时间估计（若路径拥堵，机器空转；若路径过长，导致工件延误）。
    路径规划依赖调度结果（只有确定哪些工件送去哪些机器，才知道 AGV 的起止位置）。
    AGV 并发冲突处理（MAPF 本身是 NP-hard，联合优化后更复杂）。
    全局目标通常是 Makespan 或平均延迟最小化：
