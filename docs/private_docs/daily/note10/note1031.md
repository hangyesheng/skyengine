处理一下相关机制。当前打算分别实现一下JOB和AGV层的最简单机制，分别处理。

应该可以进行如下操作：

1. AGV瞬间进行ROUTE，也就是说，在每个时间步，机器获取当前工厂的状态，并让小车根据调度的结果进行路由。
   但是当前存在一个问题：如果起始点不确定的话，小车怎么知道自己到哪里去取？读的论文中就不需要考虑这个问题，实际上小车会去起始点领取任务，根据任务要求进行运输，再回到起始点。

我们的流程：小车送完一次后，可能在任意机器的位置，如何获取下一次的任务？这个确实很抽象，不是一个很自然的场景。

这里实际上小车完成一次任务后的策略也存在很多的不同，举例来说，基本流程如下：

1. 小车完成任务 → 返回起始点（或者：小车完成任务后，直接就地等待调度中心的新任务分配。或者：工厂划分多个“AGV停靠区”，完成任务后小车返回最近的停靠点。）
2. 等待任务中心派发任务
3. 根据Job调度结果，前往取件机器
4. 完成运输任务
5. 回到起始点

调度中心计算每个AGV的：
当前可用时间（是否空闲）
当前所处位置
从当前位置到A的路径代价
然后用匹配算法（例如 Hungarian / Auction / Reinforcement Learning）为任务分配最优AGV。

operation_time可以根据调度器本身进行运算，放入到job层中处理，machine只作为起点和终点即可。


2. JOB本身的分配，只需要先遵循JSP的约束即可。分配给Machine进行。
