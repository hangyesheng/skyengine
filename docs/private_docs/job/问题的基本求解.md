# 先定义：什么是 Job-Shop Scheduling（JSS）

* 有若干作业（jobs），每个作业由一连串必须按顺序完成的工序（operations）组成。
* 每道工序需要指定的机器（machine）和处理时间（processing time）。
* 目标常见为：最小化 makespan（完工时间最大值）、最小化平均晚期、最小化总延迟等。

评估指标：**Makespan (Cmax)**、总完工时间和平均周转时间、机器闲置时间、违约/迟交惩罚等。

求解流派（常见）：

1. 规则/调度启发（dispatching rules，SPT/FCFS/LPT/EDD）——速成、在线可用。
2. 构造式启发 + 局部搜索（Giffler-Thompson 构造 / 插入式 / 交换邻域）——常用作初解与精炼。
3. 精确算法（分支定界、MILP）——小规模或需要证明最优时。
4. 元启发式（遗传、禁忌、模拟退火、粒子群）——中大规模常用，能找到高质量近似解。
5. 约束编程/CP-SAT（OR-Tools）——表达力强，实际很好用。

# 案例 1 — 用 **SPT（Shortest Processing Time）调度规则** 做贪心调度（手算/模拟）

**实例（3 个 job × 3 台机）**，每个 job 的工序序列如下（格式：机器(时间)）：

* Job 1: M1(3) → M2(2) → M3(2)
* Job 2: M2(2) → M1(1) → M3(4)
* Job 3: M1(2) → M3(1) → M2(3)

**调度策略**：模拟时间推进，每次在“可执行的工序”中选取处理时间最短（SPT），若机器空闲才执行（如无可执行则时间快进到下一个事件）。

**调度过程（关键事件）**（按时间顺序）：

1. t=0：可执行工序 = J1@M1(3), J2@M2(2), J3@M1(2)。机器空闲，选 SPT → **J2@M2(0–2)** 与 **J3@M1(0–2)**（两台不同机器可并行，优先按机器并行安排，择短先执行）。
2. t=2：M1、M2 有空。可执行：J2 下道 J2@M1(1)（因为 J2 在 M2 完成），J3 下道 J3@M3(1)，J1 仍在等待 M1。按 SPT 先执行短的：**J2@M1(2–3)**、**J3@M3(2–3)**。
3. t=3：M1 被占到 3，M3 被占到 3，M2 空。可执行：J1@M1(3)。执行 **J1@M1(3–6)**。同时 M2 有 J3 的下一工序 J3@M2(3–6)（J3 在 t=3 可进 M2），安排 **J3@M2(3–6)**。
4. t=3 之后：J2 在 M1 的第二工序后可进 M3，但 M3 此时被 J3/J1 占用，按先到先排。继续执行直到所有完成。最终 **makespan = 10**。

**按机器视图的 Gantt（时间区间为 [start, end)）**：

* M1: (0–2) J3 | (2–3) J2 | (3–6) J1
* M2: (0–2) J2 | (3–6) J3 | (6–8) J1
* M3: (2–3) J3 | (3–7) J2 | (8–10) J1

**结论**：SPT 给出一个快速可行解，makespan = **10**。优点：实现简单、速度快；缺点：不是保证最优，受 tie-break 和并发实现影响。

**何时用**：需要快速初解、或在线到达调度时（简单规则适用）。

# 案例 2 — 使用 **MILP（整数线性规划）** 建模（用于求证或给精确求解器）

同上实例可以写为 MILP（或 MIP）：

**变量**：

* (s_{j,k}) — job j 的第 k 工序的开始时间（连续，非负）。
* (C_{max}) — makespan（目标最小化）。
* 若两道在同一台机器上：二元变量 (x_{(j,k),(j',k')}\in{0,1}) 表示顺序（1 表示 (j,k) 在 (j',k') 之前）。

**约束**：

1. 顺序约束（同一 job 内）：
   [
   s_{j,k+1} \ge s_{j,k} + p_{j,k}
   ]
2. 机器互斥（若 op (a) 和 (b) 用同一机）：
   [
   s_a \ge s_b + p_b - M\cdot x_{a,b}
   ]
   [
   s_b \ge s_a + p_a - M\cdot (1-x_{a,b})
   ]
   （大 M 足够大）
3. makespan 定义：
   [
   C_{max} \ge s_{j,last} + p_{j,last}\quad \forall j
   ]

# 案例 3 — 元启发式（以 **遗传算法 GA** 为例）

当实例较大且需要高质量近似时，GA 很常用。基本思路：

1. **编码**：将一个完整调度编码为基因序列（常见是 operation-based representation：把所有 operations 列表随机排列，但顺序内含 job 的序列约束由解释器保证）。
2. **解码（解析染色体到 schedule）**：用 Giffler-Thompson 或按基因顺序插入每个 operation（当其前序完成且机器空闲时尽快安排），得到可行的 Gantt。
3. **适应度**：以 makespan（越小越好）或加权目标评估。
4. **遗传操作**：交叉（order crossover）、变异（swap/insert）、选择（轮盘或锦标赛）。
5. **局部搜索**（可选）：对每个个体做少量的邻域搜索（swap、2-opt）提升解质量。
6. **终止准则**：迭代次数或若干代无改进。

**优点**：可扩展、易与局部搜索混合。
**缺点**：参数调试（种群大小、变异率）需要经验，且不能保证全局最优。

# 实战建议（工程角度）

* **先用简单规则**（SPT / EDD / LPT / FCFS）快速得到几种不同初解，比较它们的 makespan 与机器利用率。
* **对难点用 CP 或 MIP**：小实例或需要证明最优解时用 MILP/CP。Google OR-Tools 的 CP-SAT 和 CP 模块对生产调度非常有效。
* **规模较大时**：用启发式 + 局部搜索（插入/交换/逆序）或元启发式（GA / Tabu / SA）。常见流程：规则初解 → 局部改进 → 多次随机重启 / 并行化。
* **工厂约束**：记得把额外约束建模进来（批次、换线时间、优先级、时间窗、资源共享、可用/不可用时间段）。
* **评估**：别只看 makespan，检查机器闲置分布、交付准时率、平均周转和稳定性（对扰动的鲁棒性）。
